# 快捷键自定义指南

NYTab 内置一个全局快捷键系统,支持 **序列键**(连续按下)与 **组合键**(同时按下)两种形式,可绑定聚焦搜索栏、切换搜索模式、打开 URL、跳转站内路径等动作。

## 内置快捷键

| 按键 | 类型 | 动作 | 说明 |
| --- | --- | --- | --- |
| `S` | 序列键 | `focus_search` | 聚焦搜索栏 |
| `Z` 然后 `S` | 序列键 | `toggle_search_mode` | 切换站内搜索模式(在 `mixed` 与 `bookmarks` 之间切换) |

内置快捷键 **不可删除**,也 **无法被同名自定义快捷键覆盖**(冲突检测会标红并禁用保存)。
点击设置页底部「重置为默认」可清空所有自定义快捷键,恢复到上述内置列表。

## 添加自定义快捷键

1. 进入 **设置 → 快捷键**
2. 点击「新增快捷键」
3. 在弹窗中填写:
   - **描述**:方便记忆的文字,如「打开 BiliBili」
   - **类型**:序列键 / 组合键(见下文)
   - **动作**:选择动作类型并填写参数
   - **按键**:点击录制框,按下你想要的按键
4. 冲突检测与黑名单检测 **实时进行**,若按键已被占用或属于系统级黑名单,录制框会标红并禁用「保存」
5. 点击「保存」即可立即生效,无需刷新

## 快捷键类型

### 序列键(sequence)

连续按下的一串单字符按键,**500ms 内**按完整串才触发。

- `B` → 单键序列
- `B` 然后 `L` → 双键序列(常用于 Vim 风格操作)
- `G` 然后 `I` 然后 `F` → 三键序列

缓冲区最多保留 5 个按键,超长会自动裁剪尾部。

### 组合键(combo)

修饰键 + 主键 **同时按下**。修饰键固定顺序为 `Ctrl → Win → Alt → Shift`。

- `Ctrl + K`
- `Ctrl + Shift + P`
- `Alt + /`

::: tip 💡 修饰键归一化
按下 `Control` 与 `Meta`(Mac 上是 ⌘)都会被归一化:
- `Control` → `ctrl`
- `Meta` / `Command` → `win`

所以 `Cmd+K`(Mac)和 `Win+K`(Windows)在 NYTab 里是同一个 combo。
:::

## 动作类型

| 动作 | 字段 | 说明 |
| --- | --- | --- |
| 聚焦搜索栏 | `focus_search` | 把光标移到顶部搜索栏 |
| 切换搜索模式 | `toggle_search_mode` | 在 `mixed` 与 `bookmarks` 模式之间切换(详见 [搜索引擎](/guide/search-engines)) |
| 打开 URL | `open_url` + `url` 字段 | 在新标签页打开指定 URL |
| 跳转路径 | `go_path` + `path` 字段 | 站内路由跳转,如 `/bookmarks`、`/settings/shortcuts` |

## 冲突检测

新增/编辑快捷键时,系统会实时检测:

1. **与已有快捷键冲突**:同类型(序列/组合)且按键完全相同
   - 序列键:keys 数组顺序敏感比较
   - 组合键:归一化后字符串比较(兼容 `Ctrl+K` 与 `K+Ctrl` 写法)
2. **与系统黑名单冲突**:见下文

冲突时录制框会标红,显示冲突的快捷键描述,并禁用「保存」按钮。

## 黑名单(系统级快捷键)

以下按键组合 **不允许绑定**,因为它们是浏览器/操作系统级快捷键:

| 按键 | 系统行为 |
| --- | --- |
| `Win + D` | 显示桌面 |
| `Win + L` | 锁屏 |
| `Ctrl + W` | 关闭标签页 |
| `Ctrl + Shift + Q` | 退出浏览器 |
| `Alt + F4` | 关闭窗口 |
| `F5` / `Ctrl + R` | 刷新页面 |
| `Ctrl + T` | 新标签页 |
| `Ctrl + N` | 新窗口 |
| `Ctrl + Shift + N` | 隐身窗口 |

绑定单键 `F5` 等也属于黑名单。

## 输入时禁用

当焦点在以下元素上时,**所有快捷键自动暂停触发**:

- `<input>`(包括搜索框、文本输入)
- `<textarea>`
- `contenteditable=true` 的元素(富文本编辑器)

这样在搜索栏输入关键字时,按 `S` 不会触发「聚焦搜索栏」。
录制快捷键时也会临时暂停全局监听,避免自触发。

## 示例:添加 `B + L` → 打开 BiliBili

1. 进入 **设置 → 快捷键 → 新增**
2. 填写:
   - 描述:`打开 BiliBili`
   - 类型:`序列键`
   - 动作:`打开 URL`
   - URL:`https://www.bilibili.com`
3. 点击「按键」录制框,依次按下 `B`、`L`(间隔不超过 500ms)
4. 录制框显示 `B + L`,无冲突提示
5. 保存

回到任意页面,确保焦点不在输入框内,连续按下 `B` `L`,即可在新标签页打开 BiliBili。

## 数据存储

快捷键配置存储在浏览器 `localStorage`(key:`nytab_shortcuts`)。

- **内置快捷键**:在 Pinia store `init()` 时合并到列表,不持久化到 localStorage
- **自定义快捷键**:仅持久化自定义部分,内置部分在每次加载时重新加入

这意味着同一账号在不同浏览器需要分别配置,且不会同步到后端。
若需共享配置,可手动复制 localStorage 的 `nytab_shortcuts` 字段。
